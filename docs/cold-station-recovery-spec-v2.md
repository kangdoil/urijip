# 우리집 — 콜드 스테이션 회복 스펙 v2
(순위 기반 완화 사다리 + 확정 레이아웃 매핑)

## 0. 전제

- 온보딩에서 두 사람은 각자 조건에 **1·2·3순위**를 매긴다 (기존 필수/선호/무관 대체)
- 조건 종류: 평수, 신축(년식), 인프라, 예산 — 이 중 순위를 매김
- 출퇴근 거리는 순위 대상에서 **제외** (별도 축)
- 화면은 확정된 Case 2 레이아웃 하나를 재사용한다:
  제목 → 진단 배너 → 서로 양보 바(태그+양보내용+N곳) → 동네 카드(얻는 것) → 지도 버튼

## 1. 조건 분류와 완화 방식

| 조건 | 순위 매김 | 완화 방식 | 비고 |
|---|---|---|---|
| 평수 / 신축 / 인프라 | O | **해제** (필터에서 제외) | 해제돼도 정렬 가중치로는 유지 |
| 예산 | O | **폭 넓힘만** (해제 불가) | 항상 사다리 마지막. 넓힘 한도는 예산 순위에 반비례 |
| 출퇴근 | X | **폭 넓힘만** | 순위 밖 별도 축 |

- 예산 해제 금지 이유: 해제 시 "실행 불가능한 동네"가 추천됨 → 신뢰 훼손
- 순위 조건 해제 시에도 정렬에는 반영: 같은 N곳이면 해제된 조건을 덜 어기는 동네가 위로

## 2. 기본 필터 강도 (0곳 예방)

완화 사다리 이전에, 평시 필터 자체를 순위별로 차등 적용해 0곳 발생을 줄인다.

- **1순위 = 하드 필터** — 미충족 시 탈락
- **2순위 = 소프트 필터** — 기본은 필터, 사다리에서 해제 대상
- **3순위 = 정렬 가중치** — 탈락시키지 않음, 충족 동네를 위로 올릴 뿐

※ 3순위를 처음부터 하드로 걸지 않는 것이 핵심. "3순위 = 있으면 좋고"라는
  유저의 순위 의미와 일치시키고, 콜드 스테이션 발생률 자체를 낮춘다.

## 3. 완화 사다리

0곳 발생 시, 아래 단계를 순서대로 실행하고 **결과가 처음 살아나는 단계에서 멈춘다**.
각 단계는 동일한 판정 함수를 조건 파라미터만 바꿔 재호출한다 (새 엔진 불필요).

```
0단계: 기본 필터 (2절) → 0곳이면 사다리 진입

1단계: 출퇴근 폭 확장 — 소폭 (+5분)
2단계: A·B의 2순위 해제 (예산 제외)
3단계: 출퇴근 폭 확장 — 대폭 (+15분)
4단계: 예산 폭 확장 — 각자 예산 순위에 따른 한도까지
       (예산 1순위 +0.3억 / 2순위 +0.5억 / 3순위 +0.8억)

전부 실패: 최종 0곳 상태 → 조건 조율하기 버튼 노출 (기존 팁카드 구조)
```

단계 순서 근거: 체감 부담 오름차순 — "출퇴근 5분 더"(미세 조정)가
"조건 내려놓기"(상실감)보다 가볍고, 예산은 실행 가능성에 직결되므로 항상 마지막.

- 완화는 항상 **두 사람의 같은 순위를 동시에** 내려놓는다 (대칭 = 공정)
- 두 사람의 2순위가 서로 다른 조건이어도 각자 자기 것을 내려놓으면 대칭 성립
- 조건은 사람별 독립 필터: A의 인프라 2순위 해제는 A 필터에만 영향, B가 인프라 1순위면 B 필터엔 유지
- **1순위는 어떤 단계에서도 해제하지 않는다**

## 4. 단계별 UI 매핑 (확정 레이아웃 기준)

| 단계 | 서로 양보 바 태그 | 양보 내용 표기 | 진단 배너 문구 |
|---|---|---|---|
| 0 (정상) | 태그 없음 | — | "두 분 조건이 거의 맞았어요" |
| 1 (출퇴근 소폭) | `폭 넓힘` | B +5분 등 | "출퇴근 폭을 조금 넓혀 찾아봤어요" |
| 2 (2순위 해제) | `2순위 내려놓음` | A 인프라 · B 신축 | "두 분의 2순위 조건을 잠시 내려놓고 찾아봤어요" |
| 3 (출퇴근 대폭) | `폭 넓힘` | B +15분 등 | "출퇴근 조건이 가장 멀었어요. 그만큼 폭을 넓혀 찾아봤어요" |
| 4 (예산 폭) | `예산 폭 넓힘` | A +0.5억 등 | "예산 범위를 조금 넓혀 찾아봤어요" |
| 전부 실패 | (바 흐림 처리) | — | "폭을 많이 넓혀도 찾기 어려웠어요" + 팁카드 + 조율 버튼 |

- 여러 단계가 누적된 경우(1단계 해제 + 3단계 폭) 태그는 마지막 적용 단계 기준,
  양보 내용은 누적 표기 ("A 인프라 내려놓음 · B +15분")
- 동네 카드의 "얻는 것"은 기존 로직 유지 (해당 동네가 충족하는 상위 순위 조건 표기)

## 5. 정렬 (기존 로직 확장)

```
sortScore = (통과 조건 수 × 10)
          − |A위반수 − B위반수|          // 공정 균형 (기존)
          + Σ(충족한 조건의 순위 가중치)   // 신규: 1순위 충족 +3, 2순위 +2, 3순위 +1
```

- 해제된 조건도 이 가중치 계산에는 포함 → "내려놓았는데도 그럭저럭 충족하는" 동네가 위로

## 6. 측정 (기존 지표에 추가)

- `recovery_ladder_step` 프로퍼티: 어느 단계에서 결과가 살아났는지 (0~4, fail)
- 단계별 분포로 "유저들이 실제로 어느 양보에서 회복되는지" 확인
  → 사다리 순서·폭 값 튜닝의 근거 데이터

## 7. 사다리 정지 기준 (확정)

- **1곳 이상이면 그 단계에서 멈춘다** (불필요한 추가 양보 방지)
- 단, **3곳 미만이면** 다음 단계를 한 번 더 실행해 그 결과를
  같은 화면 하단에 **"조금 더 양보하면 (+N곳)"** 접힌 섹션으로 함께 노출
  - 메인 결과(적은 양보) 우선, 추가 결과는 opt-in 열람 — 기존 접힌 보조 링크 패턴 재사용
  - 추가 섹션의 동네에는 어떤 양보가 더해졌는지 태그 표기

## 8. 폭 값 (가안 확정 — 데이터로 튜닝)

- 출퇴근: 1단계 +5분 / 3단계 +15분
- 예산: 예산 1순위 +0.3억 / 2순위 +0.5억 / 3순위 +0.8억
- 배포 후 `recovery_ladder_step` 분포를 보고 값 조정
  (특정 단계에 회복이 몰리면 그 앞 단계 폭을 키우는 식)

| 항목 | 결정 | 근거 |
|---|---|---|
| 사다리 순서 | 출퇴근 소폭 → 2순위 해제 → 출퇴근 대폭 → 예산 | 체감 부담 오름차순 ("5분 더"가 "조건 잃기"보다 가벼움) |
| 정지 기준 | 1곳이면 멈춤 + 3곳 미만 시 다음 단계 "더 보기" 병기 | 과잉 양보 방지와 화면 빈약함 보완의 절충 |
| 폭 값 | 가안으로 개발 착수, 배포 후 데이터 튜닝 | 사다리 단계별 회복 분포가 튜닝 근거 |
| 예산 완화 | 해제 불가, 폭 넓힘만, 항상 마지막 | 해제 시 실행 불가 동네 추천 → 신뢰 훼손 |
